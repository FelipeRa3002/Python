Sistema de Gerenciamento de Alunos em Python

Documentação técnica e explicação detalhada do projeto

Introdução

Este projeto consiste em um Sistema de Gerenciamento de Alunos desenvolvido em Python, com foco educacional e didático.

O objetivo principal é demonstrar domínio prático de conceitos fundamentais de programação estruturada, organização de código e manipulação de dados.

Conceitos aplicados
  • Modularização do código
  • Uso extensivo de funções
  • Manipulação de listas e dicionários
  • Estruturas condicionais e laços
  • Entrada e saída de dados pelo terminal
  • Persistência de dados em arquivos

Arquitetura do Sistema

A arquitetura do projeto é modular e segue o princípio da separação de responsabilidades.

  • corpo.py: ponto de entrada da aplicação
  • dados.py: armazenamento e persistência dos dados
  • funções.py: lógica e regras do sistema

Detalhamento do arquivo corpo.py

O arquivo corpo.py é responsável por iniciar o sistema, utilizando a estrutura if __name__ == "__main__".

Ele chama a função Menu(), transferindo o controle para o arquivo funções.py.

Detalhamento do arquivo dados.py

O arquivo dados.py funciona como a camada de dados do sistema.

Estruturas utilizadas
  • listadaTurma: lista de dicionários com dados dos alunos
  • mediadaTurmaDefinitiva: soma das médias da turma

Esse arquivo é reescrito automaticamente, garantindo a persistência dos dados.

Detalhamento do arquivo funções.py

Este arquivo representa o núcleo do sistema, contendo todas as regras de negócio.

Principais funções

limpa o terminal (ex: utiliza o comando no Windows 'cls' ou no Linux'clear').

A função limpa() é um subprograma criado para organizar a interface do usuário no terminal, tendo como único objetivo apagar todo o texto exibido anteriormente e posicionar o cursor de escrita de volta ao topo da tela. Estruturalmente, ela utiliza a palavra-chave def para declarar o bloco de código e executa internamente o comando system("cls"), que envia uma instrução direta ao sistema operacional Windows (o comando Clear Screen) para limpar o console (CMD ou PowerShell). Para que essa função opere corretamente, é necessário que a biblioteca os tenha sido previamente importada no código, sendo uma ferramenta essencial em menus interativos para manter o programa visualmente limpo e evitar o acúmulo de informações irrelevantes na tela do usuário.

Validação de dados e inserção no banco de dados.

Este trecho de código representa uma função de cadastro e processamento de dados acadêmicos, projetada para registrar o desempenho de um aluno e armazená-lo em uma lista global. A execução inicia limpando a tela para organizar a interface, solicitando em seguida o nome e as duas notas do estudante através de entradas coloridas (ANSI escape codes) para melhorar a legibilidade no terminal. Internamente, o código realiza o cálculo aritmético da média e utiliza uma estrutura condicional simplificada (if ternário) para definir a situação do aluno como "APROVADO" (em azul) ou "REPROVADO" (em vermelho), adotando o critério de nota igual ou superior a 6. O script atualiza uma variável global acumuladora (mediadaTurma) para futuros cálculos de desempenho coletivo e encapsula todas as informações individuais — nome, notas, média e situação — em um dicionário, que é então anexado à lista principal do sistema (dados.listadaTurma). O processo termina pausando a execução até que o usuário pressione "Enter", garantindo que as mensagens de confirmação sejam lidas antes de qualquer outra ação no programa.

Gerenciamento e remoção de registros.

Este trecho de código descreve uma função de gerenciamento e remoção de registros, projetada para excluir dados de alunos de forma individual ou coletiva, baseando-se em uma variável de controle x. Quando x é igual a 0, o programa executa uma exclusão seletiva: ele limpa a tela, percorre a lista principal (dados.listadaTurma) para exibir um menu numerado com os nomes dos alunos e solicita que o usuário escolha um índice; após a seleção, o código remove o aluno da lista, identifica sua média individual para subtraí-la do acumulador global mediadaTurma (mantendo o cálculo da média geral da turma atualizado) e confirma a deleção exibindo o nome do removido. Quando x é igual a 1, o código entra em um modo de exclusão total, solicitando uma confirmação do usuário; se a resposta for afirmativa (S/SIM), ele limpa completamente a lista de dados, enquanto uma resposta negativa interrompe o processo exibindo uma mensagem humorística em tom provocativo na cor lilás. O processo é finalizado com uma pausa de tela (input), garantindo que o usuário visualize o resultado da operação antes de prosseguir para o menu principal.

Edição e recalque de dados.

Este código descreve uma função de edição e recalque de dados, que permite atualizar informações específicas de um aluno já cadastrado. O processo inicia listando os nomes dos alunos numerados, onde o usuário seleciona o alvo da alteração através de um índice; em seguida, um segundo menu define se a mudança será no nome(1) ou nas notas (2) . Se o nome for escolhido, a alteração é direta no dicionário; caso uma das notas seja alterada, o código executa uma lógica de atualização de saldo, onde a média antiga é subtraída da variável global mediadaTurma, uma nova média é calculada com base no novo valor, o acumulador global é atualizado com essa nova média e a situação do aluno (APROVADO/REPROVADO) é reavaliada conforme o critério de nota 6. O script garante a integridade dos cálculos matemáticos ao converter os valores para float e finaliza pausando a tela para que o usuário confirme a conclusão da operação.

Exportação e persistência de dados

Este trecho de código descreve uma função de exportação e persistência de dados, responsável por gerar relatórios externos em dois formatos distintos baseados na variável de controle x. O script identifica automaticamente o usuário logado e a data atual, utilizando o parâmetro "txt" para criar ou atualizar um arquivo de texto legível (Relatório.txt), onde detalha o desempenho individual de cada aluno e calcula a média aritmética global da turma, classificando o rendimento coletivo como "APROVADO" ou "REPROVADO". Quando o parâmetro é "python", o código atua como um sistema de backup técnico, sobrescrevendo um arquivo dados.py com a estrutura de lista e dicionários formatada nativamente para Python, permitindo que os dados sejam importados e recuperados em execuções futuras do programa; ambos os processos incluem cabeçalhos de metadados que registram quem realizou a modificação e em qual horário exato a operação ocorreu.

Menu interativo

Este trecho de código descreve a estrutura central de controle do programa, funcionando como um menu interativo principal baseado em um laço de repetição infinito (while True). O objetivo deste bloco é gerenciar o fluxo de navegação do usuário, utilizando uma interface visual personalizada com fontes estilizadas e cores no terminal para oferecer sete opções distintas de operação. Cada opção do menu direciona o sistema para uma funcionalidade específica: as opções de 1 a 4 acionam as funções de inclusão, consulta, geração de relatório em texto e atualização de dados; as opções 5 e 6 utilizam a mesma função de exclusão, diferenciando-se pelo parâmetro (individual ou total); por fim, a opção 7 executa o salvamento automático do estado atual do sistema em formato Python antes de encerrar o programa definitivamente através do comando break.

limpa o terminal (ex: utiliza o comando no Windows 'cls' ou no Linux'clear').

Essa estrutura serve para mover um arquivo chamado dados.py da pasta de Downloads para a pasta atual do script, garantindo que o código funcione tanto no Windows quanto no Linux ou macOS. O bloco começa com o comando try, que tenta executar uma instrução do Windows: o comando move utiliza a variável de ambiente %USERPROFILE% para localizar o arquivo na pasta de Downloads do usuário e o ponto final (.) indica que ele deve ser trazido para o diretório onde o programa está rodando. Caso esse primeiro comando falhe — o que acontece se o computador não for Windows —, o Python entra no bloco except. Ali, ele tenta uma alternativa para sistemas Unix (Linux/Mac), usando o comando mv e o atalho ~ (til) para acessar a pasta pessoal do usuário. Na prática, é uma estratégia de compatibilidade entre sistemas, garantindo que o arquivo seja movido independentemente da plataforma em que o script estiver sendo executado.
